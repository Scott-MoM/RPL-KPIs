# Regional KPI Dashboard User Guide

## 1. Purpose of this guide
This guide explains how to use the Regional KPI Dashboard in a practical, non-technical way.

It is intended for users who need to:

- sign in and move around the dashboard confidently
- understand which screen to use for which task
- review KPI figures
- drill into supporting records
- run custom reports
- read or add case studies

You do not need technical knowledge to use the dashboard, but you do need to understand your role, your filters, and the difference between the main screens.

## 2. Signing in
Use this process each time:

1. Open the dashboard link.
2. Enter your email address and password.
3. If the system asks you to change your password, complete that step.
4. Wait for the main dashboard screen to load.

If you cannot sign in:

- check that you are using the correct email address
- check that the password is correct
- if needed, contact an `Admin` user for support

## 3. Roles and access
The dashboard uses role-based access. Your role controls which screens you can open and how much detail you can see.

The main roles are:

- `RPL`
- `ML` (Mountain Leader)
- `Manager`
- `Admin`
- `Funder`

In general:

- `RPL` users focus on KPI review, custom reports, and case studies
- `ML` users focus on one event at a time and case studies
- `Manager` users move between KPI, reports, ML, and funder-style views
- `Admin` users can also manage users and data operations
- `Funder` users see only the restricted funder-facing summary screen

If you cannot see a screen that someone else can see, that is often due to role permissions rather than a fault.

## 4. Main areas of the dashboard
Use `View Mode` in the left sidebar to move between screens.

Depending on your role, the available screens may include:

- `KPI Dashboard`
- `Custom Reports Dashboard`
- `Case Studies`
- `Data Request Form`
- `Admin Dashboard`
- `ML Dashboard`
- `Funder Dashboard`

Use them like this:

- `KPI Dashboard` for fast headline review
- `Custom Reports Dashboard` for tables, charts, filtering, and exports
- `Case Studies` for qualitative stories and evidence
- `Data Request Form` for formal data-access requests
- `ML Dashboard` for one-event-at-a-time operational review
- `Funder Dashboard` for restricted high-level summary reporting
- `Admin Dashboard` for user management and data maintenance

## 5. Shared controls: region, timeframe, and refresh
Most tasks depend on three shared ideas:

- region
- timeframe
- refresh status

### Region
Use `All Regions` when you want a high-level picture or when you are unsure where the records will appear.

Choose a single region when you want:

- a regional report
- a narrower chart
- a cleaner export
- a focused review

### Timeframe
Typical timeframe options include:

- `All Time`
- `Year`
- `Quarter`
- `Month`
- `Week`
- `Custom Range`

Best practice:

1. Start broader if you are unsure.
2. Confirm the records exist.
3. Narrow to the exact period you need.

### Last Data Refresh
Check `Last Data Refresh` before using any figure in:

- a meeting
- a report
- a presentation
- a support decision

The dashboard may be functioning correctly but still be too out of date for your purpose.

## 6. Using the KPI Dashboard
The `KPI Dashboard` is the fastest way to review headline figures.

Typical sections include:

- `Governance`
- `Partnerships`
- `Delivery`
- `Income`
- `Comms`
- `Case Studies`

Some sections may depend on the selected region context.

### 6.1 When to use KPI Dashboard
Use this screen when you want to:

- see the current headline picture quickly
- prepare for a meeting
- check whether a number looks right
- identify which area needs deeper follow-up

### 6.2 Step-by-step KPI workflow
1. Open `KPI Dashboard`.
2. Choose the region.
3. Choose the timeframe.
4. Choose the KPI section.
5. Review the visible KPI cards.
6. If a figure looks unusual, click the card to open drill-down.

## 7. How drill-down works
Many KPI cards can be clicked.

When you click a KPI card:

1. A detail view opens.
2. You see rows or supporting records behind the headline number.
3. You can review those records to understand what is included.
4. In some cases, you can inspect a specific selected record in more detail.

Drill-down is useful when:

- a total looks higher or lower than expected
- you want to know what sits behind a KPI
- you need confidence before sharing a figure

### Important note about participant detail
Participant-level detail depends on role and source data.

- `ML`, `Manager`, and `Admin` users may see attendee names or IDs when available
- `RPL` users should see summary-level detail only
- if source data contains only counts, even detail-access roles may see limited placeholders rather than full names

## 8. Delivery demographics
The `Delivery` area may include a demographics chart.

The chart title helps explain the source:

- `Participant Cohorts (from people type tags)` when people-tag data is available
- `Delivery Split (by event type)` when cohort tags are missing
- a fallback representation when only limited demographic data is available

Read the chart title carefully before using the result in reporting.

## 9. Custom Reports Dashboard
Use `Custom Reports Dashboard` when KPI cards are not detailed enough.

This is the main screen for:

- tables
- charts
- exportable data
- filtered analysis
- comparison work
- distance analysis

### 9.1 Available datasets
Depending on your role, the available datasets may include:

- `People`
- `Organisations`
- `Events`
- `Payments`
- `Grants`

Typical uses:

- `People` for people-record analysis where your role allows it
- `Organisations` for partnership and organisation analysis
- `Events` for delivery activity and event analysis
- `Payments` for income tracking
- `Grants` for grant pipeline and outcome reporting

### 9.2 Available output types
The available output types are:

- `Tabular`
- `Bar`
- `Line`
- `Pie`
- `UK Map`
- `Comparison Analysis`
- `Distance Analysis`

Use them like this:

- `Tabular` for row-level checking and export
- `Bar` for grouped comparison
- `Line` for trends over time
- `Pie` for share-of-total views
- `UK Map` for regional spread
- `Comparison Analysis` for direct group-versus-group comparison
- `Distance Analysis` for participant travel summaries

### 9.3 Main workflow
1. Choose the dataset or datasets you need.
2. Choose an output type.
3. Set the region and timeframe.
4. Click `Apply Report Filters`.
5. Review the initial result.
6. Open `Advanced Report Controls` if you need a narrower result.
7. Click `Apply Advanced Filters`.
8. Review the final result.
9. Export CSV if needed.

### 9.4 Understanding the two apply buttons
This screen works in two stages:

- `Apply Report Filters` updates the main report settings such as dataset, region, timeframe, and output type
- `Apply Advanced Filters` narrows the rows already returned

If you change settings and nothing happens yet, you probably still need to click the relevant button.

### 9.5 Advanced filters
Advanced controls can include:

- dataset filtering
- category filtering
- status filtering
- metric value range
- valid-date-only filtering

Use them when the first result is too broad.

### 9.6 If a report returns no rows
Check these in order:

1. Widen the region.
2. Widen the timeframe.
3. Remove advanced filters one by one.
4. If using `Line`, confirm the data includes valid dates.
5. If using `Distance Analysis`, check whether unresolved journeys are being excluded because postcode data is missing.

## 10. Case Studies
Use `Case Studies` to read and add qualitative evidence.

This screen is useful when:

- a KPI needs narrative context
- you want a real example to support a report
- you want to capture a story after an event

### 10.1 Reading case studies
1. Open `Case Studies`.
2. Apply region and date filters if needed.
3. Review the titles and dates.
4. Read the most relevant story.

### 10.2 Adding a case study
If your role allows uploads:

1. Open `Case Studies`.
2. Filter first to avoid duplicates.
3. Add a clear title.
4. Add the story text.
5. Enter the correct date.
6. Choose the correct region.
7. Submit the form.

## 11. Data Request Form
Use `Data Request Form` when you need to make a formal request for data access.

All fields are required:

- `Name`
- `Email`
- `Date`
- `Time`
- `Reason for data request`

The form is displayed on a dedicated card and uses dropdowns for `Name` and `Email`.

### 11.1 Step-by-step workflow
1. Open `Data Request Form`.
2. Choose the matching `Name` and `Email`.
3. Select the required date.
4. Select the required time.
5. Enter the reason for the request.
6. Submit the form.

### 11.2 What happens next
After you submit:

- the request is saved
- `Admin` and `Manager` users are notified in the dashboard
- your recent requests remain visible on the page

If you are an `Admin`, you can also grant temporary access permissions to a single user with an expiry date and time from the same page.

## 12. ML Dashboard
Only users with access to `ML Dashboard` can use this screen.

This screen is designed for one-event-at-a-time review.

Typical workflow:

1. Set region and timeframe.
2. Use the event selector to choose the event.
3. Review the event details.
4. Review attendee, medical, or emergency information where available and appropriate.

Always confirm the event name, date, and region before reviewing participant information.

## 13. Admin-only actions
If you are an `Admin`, you may also have access to:

- user management
- password reset handling
- manual Beacon sync
- CSV import
- audit logs
- reset or rebuild actions

These actions affect other users or the wider system. Use them carefully and only where needed.

## 14. Troubleshooting
### The page looks slow
- wait a few seconds after changing filters
- avoid very broad filters if you only need one region or one period
- refresh the browser tab if necessary

### I cannot see participant names
- names depend on source data
- some events only provide counts or partial attendee data
- some roles are not allowed to see personal detail

### I see no data
1. Check the selected region.
2. Check the selected timeframe.
3. Try `All Regions`.
4. Try `All Time`.
5. Confirm you are in the correct screen.

### A report is not changing
- click `Apply Report Filters` if you changed the main report settings
- click `Apply Advanced Filters` if you changed advanced controls

### The dashboard looks out of date
- check `Last Data Refresh`
- contact an `Admin` if the refresh looks too old

## 15. Best practice tips
- start broad, then narrow
- use KPI for quick checks
- use Custom Reports for structured analysis
- use drill-down to validate unusual figures
- use case studies to add narrative evidence
- check the refresh time before quoting a number
- use the Data Request Form when access or detail needs formal review

## 16. Need help
If something is unclear:

1. Take a screenshot.
2. Note the screen, metric, region, and timeframe.
3. Send that information to your `Admin` or support contact.
