# Mountain Leader Dashboard Manual

> Audience: `ML` users  
> Scope: event review, participant checks, and case study capture

![ML dashboard annotated guide](images/ml-dashboard-guide.svg)

## 1. About this guide
Mountain Leaders mainly use the dashboard for event-by-event work.

Available views:

- `ML Dashboard`
- `Case Studies`

ML users cannot access:

- `Admin Dashboard`
- `KPI Dashboard`
- `Custom Reports Dashboard`
- `Funder Dashboard`

## 2. Getting started
1. Sign in with your dashboard account.
2. Use `View Mode` to switch between `ML Dashboard` and `Case Studies`.
3. Set the correct region and timeframe.
4. Choose the event you want to review.
5. Check the event details before looking at participant information.

![Navigation and screen areas](images/dashboard-navigation.svg)

## 3. What the ML Dashboard is for
The ML Dashboard helps you work on one event at a time.

Use it to:

- confirm you are looking at the right event
- review event details before delivery
- check participant information where available
- review medical and emergency information where available
- capture or review case studies

## 4. Filters and event selection
Use the sidebar to choose:

- `All Regions` or a specific region
- a timeframe

Then use the `Event` selector on the page to choose the event you need.

If the event is missing:

- widen the timeframe
- switch to `All Regions`
- check whether the event has synced from Beacon

## 5. Event details
When you choose an event, the dashboard shows an `Event Details` section.

Typical fields include:

- event ID
- event name
- date
- region
- event type
- participant count
- location
- status
- description

Check this section first so you know you are in the correct event.

## 6. Participant information
If attendee data is available, the dashboard lets you review participants one at a time.

Typical workflow:

1. Choose the event.
2. Open the participant selector.
3. Choose the participant you need.
4. Review the available information carefully.

Possible sections include:

- personal information
- medical information
- emergency contact details
- participant record

Not every event will contain every field.

## 7. Medical and emergency information
The dashboard may also show event-level medical or emergency information when it is available from the source data.

Use this information carefully and only where there is a clear operational need.

## 8. Additional event information
The page may show extra event information in a readable format.

Use this when:

- a value is not shown in the main event details section
- you are checking how an event was recorded
- support staff have asked you to confirm a field

## 9. Case Studies
ML users can read and upload case studies.

This is useful for recording:

- participant feedback
- outcome stories
- short event reflections

Suggested upload steps:

1. Open `Case Studies`.
2. Check whether the story already exists.
3. Add a clear title.
4. Add the story text.
5. Choose the correct date.
6. Choose the correct region.
7. Submit the form.

## 10. Good working practice
- set the correct date range before choosing an event
- confirm the event details before reviewing participants
- treat medical and emergency information carefully
- report missing or unexpected attendee data to a Manager or Admin when needed

## 11. Troubleshooting
### No events are showing
- widen the timeframe
- switch to `All Regions`
- check that the event exists in Beacon and has synced

### No participant names are shown
- some events only include counts or partial attendee data
- this depends on the source data

### Medical or emergency details are blank
- the source record may not contain those fields
- not all events include them

### The wrong event is selected
- check the date in the event selector
- adjust the region and timeframe filters
- choose the event again after changing filters
