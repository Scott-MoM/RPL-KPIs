# RPL Dashboard Manual

> Audience: `RPL` users  
> Scope: KPI monitoring, reporting, exports, and case study management

![KPI dashboard annotated guide](images/kpi-dashboard-guide.svg)

## 1. About this guide
Regional Programme Leads use the dashboard to monitor performance, review trends, and prepare evidence for reporting.

RPL access includes:

- `KPI Dashboard`
- `Custom Reports Dashboard`
- `Case Studies`

RPL users cannot access:

- `Admin Dashboard`
- `ML Dashboard`
- personal data anywhere on the dashboard

## 2. Getting started
1. Sign in with your email address and password.
2. Use `View Mode` in the sidebar to choose the screen you need.
3. If you are not sure where the data is, start with broad filters first.
4. Narrow the filters once you know the records are present.

![Navigation and screen areas](images/dashboard-navigation.svg)

## 3. Filters and navigation
Most RPL tasks start in the left sidebar.

Main controls:

- `View Mode`
- `All Regions` or a specific region
- `Timeframe`
- `KPI Section` on the KPI page

Typical timeframe options:

- `All Time`
- `Year`
- `Quarter`
- `Month`
- `Week`
- `Custom Range`

Good practice:

- start broad
- confirm the data is there
- then narrow to the period you need

## 4. KPI Dashboard
The KPI Dashboard is the quickest way to review headline figures.

### 4.1 KPI sections
Use `KPI Section` to move between:

- `Governance`
- `Partnerships`
- `Delivery`
- `Income` when region is `Global`
- `Comms` when region is `Global`
- `Case Studies`

### 4.2 Drill-down
Click a KPI card to open the records behind the number.

Typical flow:

1. Click the KPI card.
2. Review the list of records.
3. Select a record to read more about it.

Use drill-down when:

- a number looks unexpectedly high or low
- you want to check what is included in a total
- you need confidence before sharing a figure

### 4.3 Privacy limit for RPL users
RPL users can review summary information and supporting records, but they should not see personal data anywhere in the dashboard.

This means:

- no participant names
- no participant IDs
- no postcodes
- no contact information
- no medical information

If you need personal or event-level operational detail, ask a Manager or Admin to review it in the correct view.

### 4.4 Top-line access and drill-down access
RPL users can still see top-line figures for all regions where the dashboard allows it.

Drill-down detail is limited so that:

- the summary numbers remain useful
- personal data stays protected

## 5. Custom Reports Dashboard
Use `Custom Reports Dashboard` when you need flexible analysis or an export.

![Custom reports annotated guide](images/custom-reports-guide.svg)

### 5.1 What you can do
- combine datasets
- build tables and charts
- filter by date and region
- apply advanced controls
- export results to CSV

### 5.2 Typical workflow
1. Choose the dataset or datasets you need.
2. Choose an output type.
3. Set region and timeframe.
4. Click `Apply Report Filters`.
5. Use advanced controls if needed.
6. Click `Apply Advanced Filters`.
7. Review the result and export if required.

### 5.3 Distance Analysis
Distance Analysis helps you understand how far participants travel to events.

It can show:

- number of journeys
- average, median, and maximum distance
- distance bands
- events ranked by average travel distance

Because RPL users should not see personal data, do not expect personal journey details to appear.

## 6. Case Studies
Case Studies helps you add stories alongside the numbers.

Use it to:

- read stories and testimonials
- filter by date and region
- upload a new case study

Suggested workflow:

1. Open `Case Studies`.
2. Check whether the story is already there.
3. Add a clear title.
4. Add the story text.
5. Enter the correct date.
6. Choose the correct region.
7. Submit the form.

## 7. Common RPL tasks
### Check a KPI before a meeting
1. Open `KPI Dashboard`.
2. Choose the right section.
3. Set region and timeframe.
4. Open the KPI card and review the supporting records.

### Export data for offline analysis
1. Open `Custom Reports Dashboard`.
2. Choose the dataset and output type.
3. Apply the filters.
4. Refine if needed.
5. Download the CSV.

### Add a case study to support a report
1. Open `Case Studies`.
2. Check for duplicates first.
3. Upload the new story with a clear title and date.

## 8. Troubleshooting
### No KPI data is showing
- check the region and timeframe
- try `All Regions`
- try `All Time`
- make sure you are in the correct KPI section

### The report looks unchanged
- click `Apply Report Filters`
- click `Apply Advanced Filters` if you changed advanced options

### Distance Analysis shows no rows
- make sure `Events` is included
- widen the date range
- check whether postcode data is missing

### I cannot see participant names
- that is expected for the RPL role

## 9. Good working practice
- use KPI for fast monitoring
- use Custom Reports for deeper analysis
- validate unusual numbers before sharing them
- add a case study when a report needs supporting narrative
